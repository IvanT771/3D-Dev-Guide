# Unity A* Pathfinding System

Детерминированная система поиска пути с использованием алгоритма A* для Unity, поддерживающая множественных агентов, расталкивание и очереди ожидания.

## Особенности

- ✅ **Детерминированное поведение**: Одинаковый вход всегда даёт одинаковый результат
- ✅ **Клеточная (grid-based) карта**: Движение агентов по сетке
- ✅ **Множественные агенты**: Поддержка нескольких агентов одновременно
- ✅ **Расталкивание агентов**: Избежание столкновений при попытке занять одну клетку
- ✅ **Очереди ожидания**: Эффект песочных часов в узких проходах
- ✅ **Чистый код C#**: Хорошо документированный и структурированный код
- ✅ **Визуализация**: Отображение сетки, препятствий, агентов и путей в редакторе

## Основные компоненты

### 1. SimpleAStar
Статический класс, реализующий алгоритм A* для поиска пути на сетке.

**Ключевые особенности:**
- Детерминированное разрешение конфликтов при одинаковой стоимости пути
- 4-связность для стабильного поведения
- Манхэттенская эвристика
- Поддержка проверки препятствий и занятых клеток

**Основные методы:**
```csharp
// Поиск пути от start до goal
List<Vector2Int> FindPath(Vector2Int start, Vector2Int goal, Vector2Int gridSize, 
    Func<Vector2Int, bool> isObstacle, Func<Vector2Int, bool> isOccupied = null)

// Получение соседних клеток
List<Vector2Int> GetNeighbors(Vector2Int position, Vector2Int gridSize)
```

### 2. GridManager
Singleton-компонент, управляющий сеткой, агентами и очередями ожидания.

**Функции:**
- Управление состоянием сетки и препятствий
- Регистрация и движение агентов
- Обработка столкновений и расталкивания
- Система очередей ожидания
- Визуализация в редакторе

**Основные методы:**
```csharp
// Регистрация агента на сетке
bool RegisterAgent(Agent agent, Vector2Int position)

// Попытка перемещения агента
bool TryMoveAgent(Agent agent, Vector2Int fromPosition, Vector2Int toPosition)

// Поиск пути для агента
List<Vector2Int> FindPath(Vector2Int start, Vector2Int goal, Agent requestingAgent = null)

// Установка препятствий
void SetObstacle(Vector2Int position, bool isObstacle)
```

### 3. Agent
MonoBehaviour-компонент, реализующий логику движения отдельного агента.

**Возможности:**
- Автоматический поиск пути к цели
- Плавное движение между клетками сетки
- Обработка расталкивания другими агентами
- Участие в очередях ожидания
- Автоматическое перепланирование при блокировке пути
- Визуализация пути и цели

**Основные методы:**
```csharp
// Установка цели агента
void SetTarget(Vector2Int gridTarget)
void SetTarget(Vector3 worldTarget)

// Управление состоянием
void Stop()
void Resume()

// События
System.Action<Agent> OnReachedTarget
System.Action<Agent> OnStuck
System.Action<Agent, Vector2Int> OnPositionChanged
```

### 4. DemoSceneSetup
Вспомогательный компонент для настройки демонстрационной сцены.

**Функции:**
- Автоматическое создание агентов и препятствий
- Интерактивное управление через мышь
- Обработка событий агентов
- Настройка лабиринта для демонстрации

## Использование

### Быстрый старт

1. Откройте сцену `Assets/Scenes/AStarDemo.unity`
2. Запустите сцену в Unity
3. Наблюдайте, как агенты находят пути к своим целям
4. Используйте мышь для взаимодействия:
   - **ЛКМ**: Установить новую цель ближайшему агенту
   - **ПКМ**: Добавить/удалить препятствие

### Создание собственной сцены

1. Создайте пустой GameObject и добавьте компонент `GridManager`
2. Настройте параметры сетки в инспекторе
3. Создайте агентов с компонентом `Agent`
4. Установите препятствия через `GridManager.SetObstacle()`

Пример кода:
```csharp
// Получение ссылки на GridManager
GridManager gridManager = GridManager.Instance;

// Создание агента
GameObject agentObj = new GameObject("Agent");
Agent agent = agentObj.AddComponent<Agent>();

// Установка цели
agent.SetTarget(new Vector2Int(10, 10));

// Добавление препятствия
gridManager.SetObstacle(new Vector2Int(5, 5), true);
```

### Настройка параметров

#### GridManager
- `gridSize`: Размер сетки (Vector2Int)
- `cellSize`: Размер одной клетки в юнитах
- `maxQueueSize`: Максимальное количество агентов в очереди
- `queueWaitTime`: Время ожидания в очереди
- `showGrid`: Отображение сетки в редакторе

#### Agent
- `moveSpeed`: Скорость движения агента
- `rotationSpeed`: Скорость поворота
- `pathfindingInterval`: Интервал перепланирования пути
- `stuckTimeout`: Время до определения застревания
- `showPath`: Отображение пути агента
- `showTarget`: Отображение цели агента

## Детерминированное поведение

Система обеспечивает детерминированное поведение через:

1. **Фиксированное разрешение конфликтов**: При одинаковой стоимости пути используется детерминированное сравнение координат
2. **Стабильная сортировка**: Соседние клетки всегда обрабатываются в одном порядке
3. **Целочисленные координаты**: Использование Vector2Int исключает ошибки с плавающей точкой
4. **Детерминированное семя**: Для случайных элементов (если есть) используется фиксированное семя

## Дополнительные возможности

### Расширение функционала

Система спроектирована для лёгкого расширения:

1. **Добавление весов клеток**: Модифицируйте `SimpleAStar` для поддержки различных весов передвижения
2. **Диагональное движение**: Добавьте диагональные направления в массив `directions`
3. **Различные размеры агентов**: Расширьте `GridManager` для поддержки агентов разного размера
4. **Иерархический A***: Реализуйте HPA* для больших карт

### Оптимизация производительности

Для больших карт и множества агентов:

1. Используйте пространственное разделение
2. Реализуйте кэширование путей
3. Добавьте асинхронное планирование путей
4. Оптимизируйте визуализацию через объектный пулинг

## Известные ограничения

1. 4-связность может создавать не самые короткие пути по диагонали
2. Система не поддерживает агентов разного размера
3. Отсутствует поддержка движущихся препятствий
4. Нет оптимизации для очень больших карт (>100x100)

Эти ограничения могут быть устранены при необходимости расширения системы.
