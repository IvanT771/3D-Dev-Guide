## Детальное описание компонентов

### SimpleAStar.cs

**Назначение**: Реализация алгоритма A* для поиска пути на двумерной сетке.

**Ключевые структуры данных**:
- `Node`: Структура узла с F/G/H стоимостями и позицией родителя
- `SortedSet<Node>`: Open set для эффективного извлечения узла с минимальной стоимостью
- `HashSet<Vector2Int>`: Closed set для быстрой проверки посещенных узлов
- `Dictionary<Vector2Int, Node>`: Карта узлов для быстрого доступа

**Детерминированное поведение**:
```csharp
public int CompareTo(Node other)
{
    int compare = fCost.CompareTo(other.fCost);
    if (compare == 0)
    {
        compare = hCost.CompareTo(other.hCost);
        if (compare == 0)
        {
            // Детерминированное разрешение по координатам
            compare = position.x.CompareTo(other.position.x);
            if (compare == 0)
                compare = position.y.CompareTo(other.position.y);
        }
    }
    return compare;
}
```

**Сложность алгоритма**:
- Временная: O(b^d), где b - коэффициент ветвления, d - глубина решения
- Пространственная: O(b^d) для хранения узлов в памяти

### GridManager.cs

**Назначение**: Центральный координатор системы, управляющий состоянием сетки и агентами.

**Основные структуры данных**:
```csharp
private bool[,] obstacles;                              // Матрица препятствий
private Dictionary<Vector2Int, Agent> agentPositions;   // Позиции агентов
private Dictionary<Vector2Int, Queue<Agent>> waitingQueues; // Очереди ожидания
private Dictionary<Agent, float> queueTimers;          // Таймеры очередей
```

**Система расталкивания**:
1. При попытке движения в занятую клетку
2. Поиск свободной соседней клетки для расталкивания
3. Детерминированная сортировка соседей
4. Уведомление агента о принудительном перемещении

**Система очередей**:
1. Агент добавляется в очередь при блокировке
2. Таймер ожидания предотвращает бесконечное ожидание
3. Освобождение клетки запускает обработку очереди
4. FIFO порядок обслуживания очереди

### Agent.cs

**Назначение**: Индивидуальное поведение агента, движение и взаимодействие с системой.

**Машина состояний агента**:
```
[Idle] → [Pathfinding] → [Moving] → [Waiting] → [Moving] → [Reached Target]
   ↑                        ↓           ↓
   └── [Stuck/Timeout] ←────┴───────────┘
```

**Корутины и асинхронность**:
- `MoveAlongPath()`: Основная корутина движения
- `MoveToGridPosition()`: Плавная интерполяция между клетками
- `PathfindingLoop()`: Периодическое перепланирование пути

**Обработка событий**:
```csharp
// Агент достиг цели
public System.Action<Agent> OnReachedTarget;

// Агент застрял
public System.Action<Agent> OnStuck;

// Изменение позиции
public System.Action<Agent, Vector2Int> OnPositionChanged;
```

### DemoSceneSetup.cs

**Назначение**: Автоматическая настройка демонстрационной сцены и интерактивное управление.

**Алгоритм создания лабиринта**:

2. Добавление внутренних стен для узких проходов
3. Размещение агентов в случайных валидных позициях

**Интерактивность**:
- Обработка кликов мыши через Raycast
- Динамическое изменение препятствий
- Установка новых целей агентам

## Алгоритмические особенности

### Детерминированность

**Проблемы с детерминизмом**:
1. Порядок обработки узлов с одинаковой стоимостью
2. Порядок обработки агентов
3. Разрешение конфликтов при расталкивании
4. Выбор соседних клеток

**Решения**:
1. Детерминированное сравнение узлов по координатам
2. Фиксированный порядок обновления агентов
3. Стабильная сортировка соседних клеток
4. Использование целочисленных координат

### Оптимизации производительности

**A* алгоритм**:
- `SortedSet` для эффективного извлечения минимального элемента
- `HashSet` для быстрой проверки принадлежности closed set
- Ранний выход при достижении цели

**Управление агентами**:
- Ленивое обновление очередей
- Кэширование валидных позиций
- Пакетная обработка событий

**Визуализация**:
- Отрисовка только в режиме редактора
- Условная отрисовка (флаги showGrid, showPath)
- Переиспользование LineRenderer компонентов

```csharp
{
    private Vector2Int lastPosition;
    
    void Update()
    {
        Vector2Int currentPos = GridManager.Instance.WorldToGrid(transform.position);
        if (currentPos != lastPosition)
        {
            GridManager.Instance.SetObstacle(lastPosition, false);
            GridManager.Instance.SetObstacle(currentPos, true);
            lastPosition = currentPos;
        }
    }
}
```

## Конфигурация и настройка

### Рекомендуемые параметры

**Для маленьких карт (≤20x20)**:
- pathfindingInterval: 0.3s
- maxQueueSize: 2
- queueWaitTime: 0.3s

**Для средних карт (20x20 - 50x50)**:
- pathfindingInterval: 0.5s
- maxQueueSize: 3
- queueWaitTime: 0.5s

**Для больших карт (≥50x50)**:
- pathfindingInterval: 1.0s
- maxQueueSize: 5
- queueWaitTime: 1.0s
